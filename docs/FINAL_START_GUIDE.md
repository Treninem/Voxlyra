# Вокслира — финальная инструкция запуска через GitHub → Bothost

Эта инструкция сделана для запуска проекта через GitHub и Bothost.

## 1. Создать Telegram-бота

1. Откройте Telegram.
2. Найдите `@BotFather`.
3. Нажмите Start.
4. Введите `/newbot`.
5. Укажите название бота, например `Вокслира`.
6. Укажите username бота, например `voxlyra_bot`.
7. BotFather выдаст токен. Он нужен для переменной `BOT_TOKEN`.

Важно: токен никому не отправлять и не выкладывать на GitHub.

## 2. Узнать свой Telegram ID

1. Откройте Telegram.
2. Найдите бота `@userinfobot` или похожего бота для просмотра ID.
3. Нажмите Start.
4. Скопируйте свой числовой ID.
5. Этот ID вставляется в `OWNER_IDS`.

Можно несколько владельцев через запятую:

```env
OWNER_IDS=111111111,222222222
```

## 3. Создать канал для публикаций

1. В Telegram создайте канал.
2. Придумайте название, например `Вокслира | Новые истории`.
3. Сделайте публичный username канала, например `@voxlyra_news`.
4. Добавьте бота в канал.
5. Дайте боту право публиковать сообщения.
6. В переменную `CHANNEL_ID` вставьте username канала:

```env
CHANNEL_ID=@voxlyra_news
```

## 4. Подготовить GitHub

1. Откройте GitHub.
2. Создайте новый репозиторий.
3. Название можно поставить `voxlyra_bot`.
4. Репозиторий можно сделать private.
5. Распакуйте архив `voxlyra_bot_v1.8.0_cross_flow_stage8.zip`.
6. Загрузите содержимое папки `voxlyra_bot` в репозиторий.
7. Не загружайте `.env`, если создавали его у себя локально.
8. Файл `.env.example` можно загружать.

## 5. Создать проект на Bothost

1. Откройте Bothost.
2. Создайте новый проект.
3. Выберите запуск из GitHub/GitLab-репозитория.
4. Подключите репозиторий с ботом.
5. Укажите ветку, обычно `main`.
6. Если Bothost спросит способ запуска, выберите Dockerfile или Python-проект.

В проекте уже есть:

```text
main.py
requirements.txt
Dockerfile
.env.example
```

## 6. Добавить переменные окружения на Bothost

Вставьте такие переменные:

```env
BOT_TOKEN=токен_от_BotFather
OWNER_IDS=ваш_Telegram_ID
DATABASE_PATH=data/voxlyra.sqlite3
RUN_WEBAPP=true
PORT=3000
WEBAPP_URL=https://адрес_проекта_на_bothost
CHANNEL_ID=@username_канала
BOT_USERNAME=username_бота_без_@
PROJECT_NAME=Вокслира
PUBLIC_VERSION_VISIBLE=false
PROJECT_VERSION=v1.8.0-cross-flow-stage8
```

## 7. Где взять WEBAPP_URL

`WEBAPP_URL` не создаётся вручную заранее. Он появляется после публикации проекта на Bothost.

Порядок:

1. Создайте проект на Bothost.
2. Подключите GitHub.
3. Запустите деплой.
4. Откройте настройки проекта.
5. Найдите раздел с публичным адресом, доменом или web-приложением.
6. Скопируйте HTTPS-адрес вида:

```text
https://ваш-проект.bothost.app
```

7. Вставьте его в `WEBAPP_URL` без слеша в конце.

Правильно:

```env
WEBAPP_URL=https://voxlyra.bothost.app
```

Неправильно:

```env
WEBAPP_URL=https://voxlyra.bothost.app/
```

Если Bothost не показывает адрес, проверьте:

```env
RUN_WEBAPP=true
PORT=3000
```

В коде веб-часть слушает `0.0.0.0:3000`, это уже настроено.

## 8. Запустить деплой

1. Нажмите Deploy/Запустить.
2. Дождитесь сборки.
3. Откройте логи.
4. Если ошибок нет, бот должен начать отвечать в Telegram.

## 9. Проверить бота

1. Откройте своего бота.
2. Нажмите Start.
3. Должно появиться меню:

```text
📚 Читать
🎧 Слушать
⭐ Моё
➕ Ещё
```

4. Если ваш ID указан в `OWNER_IDS`, появится скрытая кнопка:

```text
👑 Управление
```

5. Откройте:

```text
👑 Управление → 🧩 Система
```

Там будут проверки переменных и подсказки, что не заполнено.

## 10. Проверить Mini App

1. Нажмите `📚 Читать`.
2. Должен открыться каталог.
3. Если не открывается, проверьте `WEBAPP_URL`.
4. Если платные главы не открываются внутри Mini App, проверьте `BOT_USERNAME` и `BOT_TOKEN`.

## 11. Проверить автора

1. Нажмите `➕ Ещё`.
2. Нажмите `✍️ Автору`.
3. Нажмите `Стать автором`.
4. Заполните профиль.
5. Добавьте книгу.
6. Выберите галочками жанры, теги, аудиторию и предупреждения.
7. Добавьте главы.
8. Отправьте книгу на проверку.

## 12. Проверить модерацию

1. Откройте `👑 Управление`.
2. Откройте `👥 Администрация`.
3. Добавьте пользователя по ID или username.
4. Включите ему только нужные права.
5. У модератора появится только разрешённое меню.

## 13. Проверить канал

1. Убедитесь, что бот администратор канала.
2. Опубликуйте книгу через модерацию.
3. Если `CHANNEL_ID` указан правильно, бот попробует отправить пост в канал.

## 14. Если бот не запускается

Проверьте логи Bothost.

Частые ошибки:

- неверный `BOT_TOKEN`;
- не указан `OWNER_IDS`;
- `WEBAPP_URL` со слешем в конце;
- бот не добавлен в канал;
- база указана не в `data/`;
- порт отличается от `PORT=3000`;
- репозиторий загружен не из папки `voxlyra_bot`, а уровнем выше.

## 15. Что нельзя загружать в GitHub

Не загружать:

```text
.env
*.sqlite3
токены
личные реквизиты
резервные копии базы
```

Загружать можно:

```text
.env.example
requirements.txt
Dockerfile
main.py
app/
templates/
static/
docs/
tests/
```
