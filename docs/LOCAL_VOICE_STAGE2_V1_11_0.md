# VoxLyra v1.11.0 — этап 2: автономный русский голос

В рабочую ветку встроен локальный `Vosk-TTS` с русской многоголосой моделью
`vosk-model-tts-ru-0.9-multi`. Отдельный сервер, API-ключ, адрес
`TTS_MOSS_URL` и ручная загрузка голосов владельцу не требуются.

## Автоматическая установка

- Docker устанавливает `vosk-tts`;
- русская модель загружается во время сборки образа;
- модель хранится внутри образа и не скачивается при каждом запуске;
- Piper остаётся только аварийным резервом.

## Автоматический выбор голосов

При первом запуске VoxLyra создаёт тестовые записи всех пяти голосов модели.
Для каждой записи проверяются:

- целостность аудиофайла;
- длительность относительно текста;
- громкость и перегруз;
- доля и максимальная длина тишины;
- скорость генерации.

Из женской и мужской групп автоматически выбирается наиболее стабильный голос.
Результат хранится в `storage/tts/vosk_voice_profile.json`, поэтому повторная
проверка при каждом перезапуске не выполняется.

Автоматическая оценка определяет техническую стабильность, а не художественную
красоту тембра. Поэтому владелец может прослушать созданные образцы и вручную
выбрать женский и мужской голос в разделе «Управление → Озвучивание».

## Честная диагностика владельца

В центре управления добавлен отдельный раздел «Озвучивание». В нём видны:

- все фактически доступные движки;
- сработал ли Vosk или резервный Piper;
- выбранные номера мужского и женского голосов;
- результаты проверки пяти голосов;
- образцы для прослушивания;
- состояние очереди синтеза;
- количество завершённых, ошибочных и повторно использованных заданий.

Владелец может повторно запустить проверку и заменить любой из двух голосов.
Обычные пользователи техническую диагностику не видят.

## Порядок движков

Обычный режим: `Vosk → MOSS → Qwen → Piper`.
Режим повышенного качества: `MOSS → Qwen → Vosk → Piper`.

Пустые адреса внешних движков больше не означают немедленный возврат к Piper:
локальный Vosk находится перед ним.

## Новые настройки

- `TTS_VOSK_AUTO_SELECT=true`;
- `TTS_VOSK_FEMALE_CANDIDATES=0,1,2`;
- `TTS_VOSK_MALE_CANDIDATES=3,4`;
- `TTS_VOSK_PROFILE_PATH=storage/tts/vosk_voice_profile.json`;
- `TTS_VOSK_BENCHMARK_TIMEOUT_SECONDS=180`.

Менять их владельцу не требуется: значения уже включены в готовую сборку.

## Проверка этапа

- полный pytest: **167 успешно, 1 пропущен**;
- отдельные тесты локального голоса и автовыбора: **10/10**;
- Python и JavaScript проходят синтаксическую проверку.

Фактическая приёмка тембра на телефоне выполняется после финального Redeploy.

движка; скрытый возврат к прежнему голосу не должен остаться незаметным.
