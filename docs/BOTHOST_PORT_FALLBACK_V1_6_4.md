# VoxLyra v1.6.4 — Bothost port fallback

Исправление для ситуации, когда приложение внутри контейнера запускается, но домен Bothost возвращает Bad Gateway / Gateway Timeout и в логах нет `GET /health`.

Что изменено:
- Dockerfile теперь объявляет оба порта: `8080` и `3000`.
- Код продолжает слушать порт из переменной `PORT`.

Как проверять:
1. Сначала используйте стандартно: `PORT=8080`, порт веб-приложения Bothost `8080`.
2. Если домен всё равно не доходит до приложения, временно поставьте в Bothost и в переменных: `PORT=3000`, порт веб-приложения `3000`.
3. После изменения порта нужен именно Redeploy / пересборка.
4. В логах должно быть `Uvicorn running on http://0.0.0.0:3000` или `8080` — в зависимости от выбранного порта.
5. При открытии `/health` в логах должна появляться строка `GET /health`.

Если `GET /health` не появляется, запрос не доходит до контейнера, и причина в проксировании домена на стороне хостинга.
