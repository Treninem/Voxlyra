# Полная инструкция GitHub → Bothost

## Что понадобится

1. Telegram-бот, созданный через BotFather.
2. GitHub-аккаунт.
3. Bothost-аккаунт.
4. Telegram ID владельца.
5. Telegram-канал для автопостинга.

## Шаг 1. Распаковать архив

Распакуйте архив `voxlyra_bot_v1.8.0_cross_flow_stage8.zip`.

Внутри будет папка:

```text
voxlyra_bot/
```

Именно содержимое этой папки нужно загрузить в GitHub.

## Шаг 2. Создать репозиторий GitHub

1. Откройте GitHub.
2. Нажмите `New repository`.
3. Название можно поставить `voxlyra_bot`.
4. Выберите `Private`, если не хотите показывать код публично.
5. Создайте репозиторий.
6. Загрузите файлы из папки `voxlyra_bot/`.

Не загружайте файл `.env`, если создавали его локально. В проекте есть только `.env.example`.

## Шаг 3. Создать проект на Bothost

1. Откройте Bothost.
2. Создайте новый проект.
3. Выберите деплой из GitHub.
4. Подключите репозиторий `voxlyra_bot`.
5. Укажите ветку, обычно `main`.
6. Тип запуска: Python/Docker, в зависимости от панели Bothost.

В проекте уже есть:

```text
Dockerfile
requirements.txt
main.py
```

## Шаг 4. Вставить переменные окружения

В Bothost откройте переменные окружения и вставьте:

```env
BOT_TOKEN=токен_от_BotFather
BOT_USERNAME=username_бота_без_@
OWNER_IDS=ваш_Telegram_ID
DATABASE_PATH=data/voxlyra.sqlite3
RUN_WEBAPP=true
PORT=3000
WEBAPP_URL=https://адрес_проекта_на_Bothost
CHANNEL_ID=@username_канала
PROJECT_NAME=Вокслира
PUBLIC_VERSION_VISIBLE=false
PROJECT_VERSION=v1.8.0-cross-flow-stage8
```

Если `WEBAPP_URL` ещё неизвестен, сначала можно временно поставить:

```env
WEBAPP_URL=https://example.bothost.app
```

После первого запуска Bothost покажет настоящий публичный адрес. Его нужно скопировать и заменить `WEBAPP_URL`.

## Шаг 5. Первый деплой

1. Запустите деплой.
2. Дождитесь успешного запуска.
3. Найдите публичный HTTPS-адрес проекта.
4. Вставьте его в `WEBAPP_URL`.
5. Перезапустите проект.

## Шаг 6. Проверить Mini App

Откройте в браузере:

```text
WEBAPP_URL/health
```

Пример:

```text
https://voxlyra.bothost.app/health
```

Если открывается JSON-ответ, веб-часть работает.

Потом откройте:

```text
WEBAPP_URL/readiness
```

Там будут безопасные проверки окружения без показа токена.

## Шаг 7. Проверить бота

1. Откройте своего бота в Telegram.
2. Нажмите `/start`.
3. Проверьте главное меню.
4. Если ваш Telegram ID указан в `OWNER_IDS`, должна появиться кнопка `👑 Управление`.
5. Откройте `👑 Управление → 🧩 Система`.
6. Проверьте, какие пункты отмечены зелёными галочками.

## Шаг 8. Подключить канал

1. Создайте Telegram-канал.
2. Загрузите аватар `voxlyra_channel_avatar_final.png`.
3. Добавьте бота в канал.
4. Выдайте боту право публиковать сообщения.
5. В `CHANNEL_ID` укажите `@username_канала`.
6. Перезапустите проект.

## Шаг 9. Проверить автора

1. В боте нажмите `➕ Ещё`.
2. Нажмите `✍️ Автору`.
3. Зарегистрируйте профиль автора.
4. Создайте книгу.
5. Добавьте главы.
6. Отправьте книгу на проверку.

## Шаг 10. Проверить модерацию

1. В скрытом меню владельца добавьте администратора.
2. Выдайте ему только нужные права.
3. Проверьте, что у него показываются только разрешённые разделы.

## Шаг 11. Проверить покупку Stars

Для реальной проверки Stars нужен запущенный бот и корректная настройка BotFather/Telegram. В тестовом локальном режиме можно проверить только внутреннюю логику покупок, но не реальный платёж Telegram.

## Шаг 12. Что делать при ошибке

Смотреть в порядке:

1. Логи Bothost.
2. `👑 Управление → 🧩 Система`.
3. `WEBAPP_URL/health`.
4. `WEBAPP_URL/readiness`.
5. Переменные окружения.
6. Права бота в канале.
7. Правильно ли указан `OWNER_IDS`.
