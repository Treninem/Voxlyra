# Сквозная проверка — v1.8.0

## Права и безопасность

Владельческие действия отделены от делегируемых прав. Администратору можно выдавать только рабочие права модерации, жалоб, возвратов, рекламы, статистики и поддержки. Выплаты, комиссия, полное удаление, управление администрацией и настройки платформы остаются только у владельца.

Блокировка пользователя теперь действует во всех точках входа:

- сообщения и кнопки Telegram;
- предварительная проверка оплаты Stars;
- Mini App API;
- личная библиотека и платный доступ.

## Mini App

- Telegram `initData` требует корректную дату и подпись;
- слишком старая или будущая сессия отклоняется;
- платная глава не включается в HTML до проверки доступа;
- публичная диагностика показывает только готовность, без структуры окружения;
- `/docs`, `/redoc` и `/openapi.json` отключены;
- служебная версия не попадает в исходный код публичных страниц;
- скрытые элементы не могут проявиться из-за CSS.

## Сквозные сценарии

Автоматически проверяются:

1. Публичное открытие каталога, книги, аудио и настроек.
2. Отказ приватного API без Telegram-сессии.
3. Закрытая платная глава до покупки.
4. Открытие той же главы после зарегистрированной покупки.
5. Запрет изменения чужой книги другим автором.
6. Панель владельца и ограниченная панель модератора.
7. Запрет выплат для модератора.
8. Немедленная потеря доступа после блокировки читателя.
9. Неделегируемость владельческих прав, включая старые записи базы.

## Bothost

Рабочие параметры не изменились:

```env
RUN_WEBAPP=true
PORT=3000
DATABASE_PATH=data/voxlyra.sqlite3
```

При обновлении сохранять рабочие `.env` и `data/voxlyra.sqlite3`, затем выполнять Redeploy.
