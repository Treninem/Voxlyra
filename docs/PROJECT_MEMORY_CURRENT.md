## Публикация и проверка

- Любой путь загрузки книги использует единый сервис завершения: Telegram, Mini App, ручная глава и импорт файла.
- Владелец публикуется сразу, если нет однозначного технического/антиспам-нарушения или неподтверждённой копии.
- Книга обычного автора проходит предварительную проверку и передаётся человеку, чтобы автоматическая система не пропустила нарушение и не заблокировала хорошую книгу.
- Уведомления получают владелец и сотрудники с правом `mod_books`.
- Первое напоминание через 6 часов, повторные через 12 часов; после решения напоминания прекращаются.
- При возврате на доработку автор получает причину и кнопку открытия книги.
- Опубликованная книга появляется в каталоге и отправляется в канал.
- Пост канала содержит обложку, прямую ссылку и кнопку открытия книги.

## Обложки

- Принимать фотографию или файл JPG/JPEG/PNG/WEBP.
- Сохранять Telegram `file_id` и локальный файл `storage/covers/<book_id>.<ext>`.
- После Redeploy восстанавливать отсутствующие файлы из Telegram.
- Во всех публичных карточках всегда пробовать `/media/cover/<book_id>` и только при реальной ошибке показывать буквенную заглушку.
- В кабинете автора использовать защищённый адрес `/api/author/book/<id>/cover`.
- В канал сначала отправлять обложку по `file_id`, затем локальный файл.

## Читалка

- Процент считать только внутри текста главы, без заголовка, рекламы и комментариев.
- В настройках читалки есть ввод номера главы и быстрый переход.
- При отсутствии номера показывать доступный диапазон.

## Ранее закреплённое

- Проверка копий: нормализованное название, автор, близкое совпадение и SHA-256 файла.
- Поиск Mini App: части названия, точное название, похожие варианты и небольшие опечатки.
- Владелец повторно публикует бесплатно; платное продвижение одной книги — не чаще раза в 30 дней.
- В многошаговых сценариях использовать «Назад» или «Отмена» по смыслу.

## Сохранение данных при обновлении

Всегда сохранять `.env`, `data/voxlyra.sqlite3` и всю папку `storage/`.
