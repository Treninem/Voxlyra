# Рабочая память проекта VoxLyra

## Текущая каноническая точка

Финальная версия: **VoxLyra v1.11.1**. Это исправленная финальная сборка поверх v1.11.0.

Пользователь должен получить три обязательных файла: обновление v1.11.0 → v1.11.1, полный резервный архив и отдельный архив переноса/рабочей памяти.

## Ключевые правила

### Бесплатные книги и главы

- Цена всей книги `0 Stars` означает полностью бесплатную книгу.
- Все существующие и будущие главы такой книги бесплатны.
- У бесплатной главы запрещены экран «Глава закрыта», цена `0 Stars`, рублёвый эквивалент и кнопка покупки.
- Вне Telegram при защищённом тексте показывается только «Откройте главу в Telegram» без платёжных элементов.
- При миграции старые противоречивые главы нормализуются, пакеты текстовых глав отключаются, прежние цены сохраняются только как неактивный черновик.

### Согласия

- Согласия хранятся в базе по пользователю и документу.
- Повторный вход, Redeploy и обычное обновление не требуют повторного подтверждения.
- Изменение технического хэша PDF не аннулирует согласие.
- Повторное принятие требуется только для явно отмеченной существенной версии через `legal_reaccept_<code>_version`.
- Документы автора отделены от базовых пользовательских документов.

### Версия

- Номер и название сборки показываются только владельцу в защищённом центре управления и диагностике.
- Пользователям, авторам и обычным модераторам версия не показывается.
- `OWNER_BUILD_VERSION = v1.11.1`; версия статических файлов также обновлена для сброса Telegram-кэша.

## Всё сохранённое из v1.11.0

- локальный Vosk-TTS и резервный Piper;
- сегментное непрерывное озвучивание и подготовка следующей главы;
- раздельные цены всей книги и отдельных глав;
- рекомендации «Для вас»;
- комментарии, ответы, спойлеры, реакции и жалобы;
- помощник по книге без будущих глав;
- аналитика автора, достижения, уведомления и карточки цитат;
- Premium в Telegram Stars.

## Нельзя нарушать

- Перед обновлением сохранять `.env`, `data/voxlyra.sqlite3` и всю папку `storage`.
- Не сбрасывать базу, книги, главы, покупки, доходы, комиксы и статистику.
- Не менять аватары бота и канала.
- Не показывать публично номер рабочей сборки и внутреннюю диагностику.
- ЮKassa не активировать без отдельного решения владельца.
- После следующего обновления снова создать отдельный архив переноса.
