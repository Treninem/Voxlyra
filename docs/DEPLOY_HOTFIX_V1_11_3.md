# VoxLyra v1.11.3 — исправление Redeploy

Причины исправления:

1. Предыдущий небольшой архив v1.11.1 → v1.11.2 нельзя устанавливать непосредственно на v1.10.5.
2. Загрузка большой Vosk-модели выполнялась внутри Docker-сборки и могла надолго блокировать Redeploy.

Теперь обновление накопительное, а модель загружается после запуска в постоянную папку `storage`.
