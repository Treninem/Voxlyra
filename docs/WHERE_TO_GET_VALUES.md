# Где брать значения для Bothost

Этот файл нужен, чтобы не путаться, какие данные где брать перед запуском.

## BOT_TOKEN

Где взять:
1. Откройте Telegram.
2. Найдите `@BotFather`.
3. Напишите `/newbot`.
4. Укажите имя бота.
5. Укажите username бота, который заканчивается на `bot`.
6. BotFather выдаст токен вида `123456789:AA...`.

Куда вставлять:

```env
BOT_TOKEN=токен_от_BotFather
```

Не публикуйте токен в GitHub.

## BOT_USERNAME

Где взять:
- это username бота, который вы создали через BotFather.

Пример: если бот `@voxlyra_reader_bot`, то вставлять нужно без `@`:

```env
BOT_USERNAME=voxlyra_reader_bot
```

## OWNER_IDS

Где взять:
1. Откройте Telegram.
2. Найдите бота `@userinfobot` или похожего бота для просмотра ID.
3. Узнайте свой Telegram ID.
4. Вставьте его в Bothost.

Пример:

```env
OWNER_IDS=123456789
```

Если владельцев несколько:

```env
OWNER_IDS=123456789,987654321
```

## WEBAPP_URL

Это публичный HTTPS-адрес проекта на Bothost.

Его нельзя взять до первого деплоя. Порядок такой:

1. Загрузите проект в GitHub.
2. Создайте проект на Bothost.
3. Подключите GitHub-репозиторий.
4. Запустите деплой.
5. Откройте проект на Bothost.
6. Найдите раздел с доменом/публичным URL.
7. Скопируйте адрес вида `https://...`.
8. Вставьте его в переменную `WEBAPP_URL`.

Пример:

```env
WEBAPP_URL=https://voxlyra.bothost.app
```

Важно: без слеша в конце.

Правильно:

```env
WEBAPP_URL=https://voxlyra.bothost.app
```

Неправильно:

```env
WEBAPP_URL=https://voxlyra.bothost.app/
```

## CHANNEL_ID

Если канал публичный, можно указать username канала:

```env
CHANNEL_ID=@voxlyra_news
```

Важно:
1. Добавьте бота в канал.
2. Сделайте его администратором.
3. Разрешите публикацию сообщений.

Если канал закрытый, нужен числовой ID канала. Его можно получить через служебные боты или временно сделать канал публичным, настроить, потом вернуть приватность.

## DATABASE_PATH

Для Bothost оставляем так:

```env
DATABASE_PATH=data/voxlyra.sqlite3
```

Так база лежит в папке `data/` и не смешивается с кодом.

## RUN_WEBAPP

Для Mini App должно быть:

```env
RUN_WEBAPP=true
```

Если выключить, бот будет работать без каталога, читалки и аудиоплеера.

## PORT

Оставляем:

```env
PORT=3000
```

В коде веб-приложение слушает `0.0.0.0:3000`, чтобы Bothost увидел Mini App снаружи.
