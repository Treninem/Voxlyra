# VoxLyra v1.10.5 — этап 4 из 5

## Непрерывный сегментный плеер

Этап 4 заменяет ожидание цельного MP3 на очередь коротких звуковых фрагментов.

Реализовано:

- два чередующихся HTML-аудиоэлемента: один воспроизводит, второй заранее буферизует следующий фрагмент;
- первый звук запускается после готовности первого короткого сегмента, без ожидания всей главы;
- сервер готовит окно следующих сегментов с приоритетом текущей главы;
- переход между сегментами выполняется без повторного открытия плеера;
- на последнем сегменте заранее создаётся сессия следующей главы;
- первый фрагмент следующей главы загружается в свободный аудиобуфер и включается сразу после окончания текущей;
- запасной переход создаёт следующую сессию обычным способом, если предварительная загрузка не успела;
- скорость меняется в браузере одновременно у обоих буферов и больше не требует повторного синтеза;
- сохранение позиции хранит номер сегмента, время внутри сегмента и общую позицию;
- перемотка ±15 секунд работает по общей шкале главы;
- Media Session поддерживает play/pause, перемотку, предыдущую и следующую главы;
- при слабом интернете окно сегментов запрашивается повторно, повреждённый URL обновляется;
- готовые сегменты сохраняются в Cache Storage устройства по хэшу текста;
- для служебного модераторского доступа постоянный кэш отключён;
- добавлен собственный прогресс-бар главы, а не длительность одного фрагмента.

## Серверная основа

В рабочую ветку включены:

- `tts_text.py` — смысловая нарезка;
- `tts_providers.py` — MOSS/Qwen через приватный сервер и локальный Piper как резерв;
- `tts_queue.py` — приоритетная очередь и устранение повторных заданий;
- `tts_sessions.py` — временные защищённые сессии и подписанные ссылки на фрагменты;
- новые API создания, чтения и закрытия TTS-сессий;
- повторная проверка доступа к платной главе при выдаче каждого фрагмента.
