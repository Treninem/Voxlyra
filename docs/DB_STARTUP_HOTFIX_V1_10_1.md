# Voxlyra v1.10.1 — исправление параллельного запуска базы

## Причина ошибки

`run_bot()` и lifespan FastAPI стартуют параллельно и оба вызывают `init_db()`. На существующей SQLite-базе два подключения могли одновременно прочитать `PRAGMA table_info(books)`, не увидеть `content_type` и выполнить одинаковое добавление столбца. Первое завершалось успешно, второе падало с `duplicate column name: content_type`.

## Исправление

- Добавлена блокировка инициализации базы внутри процесса.
- Подготовленные пути баз запоминаются отдельно, поэтому тестовые и дополнительные базы не смешиваются.
- Все мягкие операции добавления столбцов выполняются через защищённый DDL-помощник.
- Игнорируется только точная гонка одинакового `ADD COLUMN`; другие ошибки SQLite пробрасываются.
- Повторный запуск, частично выполненная предыдущая миграция и обновление поверх существующей базы безопасны.

Пользовательские данные не изменяются и не очищаются.
