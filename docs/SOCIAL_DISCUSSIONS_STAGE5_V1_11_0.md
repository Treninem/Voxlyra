# VoxLyra v1.11.0 — этап 5: обсуждения и реакции к главам

## Что реализовано

- Комментарии относятся к конкретной текстовой главе и доступны только читателю, который может открыть эту главу.
- На комментарий можно ответить. Ответы визуально группируются под исходным сообщением.
- Глубина ветки ограничена одним уровнем: ответ на ответ прикрепляется к корневому комментарию, поэтому обсуждение не превращается в узкую бесконечную лестницу.
- Комментарий можно заранее отметить как спойлер. Его текст скрыт до явного нажатия читателя.
- Каждый читатель может поставить или убрать один лайк у комментария.
- На главу можно выбрать одну из шести реакций: захватывающе, понравилось, тронуло, смешно, неожиданно, эпично. Повторное нажатие снимает реакцию, выбор другой заменяет предыдущую.
- На комментарий можно отправить жалобу с причиной. Повторная активная жалоба того же пользователя не создаёт дубликат.
- Нельзя пожаловаться на собственный комментарий.

## Модерация

- В очереди комментариев показываются активные жалобы, отметка спойлера, число лайков и тип сообщения: основной комментарий или ответ.
- Жалоба на комментарий содержит прямую кнопку перехода к карточке этого комментария.
- После скрытия комментария все его активные жалобы автоматически закрываются.
- Скрытый корневой комментарий не оставляет в публичном интерфейсе осиротевшие ответы.
- Сохраняется журнал действий модератора и уведомление автору скрытого материала.

## Новая структура данных

- В `comments` добавлены `parent_id` и `is_spoiler`.
- `comment_likes` хранит уникальный лайк пользователя к комментарию.
- `chapter_reactions` хранит один текущий выбор пользователя для главы.
- Для жалоб используется существующая таблица `complaints` с `target_type='comment'`.
- Миграции выполняются автоматически и не сбрасывают старые комментарии, книги, главы, покупки или `storage`.

## API Mini App

- `GET/POST /api/reader/{chapter_id}/comments`
- `POST /api/comments/{comment_id}/like`
- `POST /api/comments/{comment_id}/report`
- `GET/POST /api/reader/{chapter_id}/reactions`

Все действия требуют действительный Telegram initData и проверку доступа к главе.
