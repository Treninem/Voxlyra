# VoxLyra v1.13.12.1 — установка hotfix импорта

## Что исправлено
- импорт больше не удерживает обработчик Telegram до завершения всего ZIP;
- после загрузки архив ставится в постоянную фоновую очередь;
- бот остаётся доступным во время проверки, распаковки и импорта книг;
- сообщение импорта показывает этап, количество обработанных книг, процент, добавленные книги, дубли и ошибки;
- задания выполняются последовательно, поэтому несколько ZIP не перегружают сервер одновременно;
- незавершённое задание после перезапуска безопасно очищает только свой неполный пакет и запускается заново;
- старое заводское ограничение «500 книг» один раз переводится в режим «без ограничения»;
- пользовательские значения, отличные от старого значения 500, сохраняются;
- обычное поле `id` в пакетах больше не принимается за ID книги VoxLyra: для точной замены используется `voxlyra_book_id` или `existing_book_id`;
- экспорт VoxLyra теперь записывает `voxlyra_book_id`.

## Установка
1. Сохранить текущие `.env`, `data/` и `storage/`.
2. Распаковать архив поверх файлов проекта с заменой.
3. Не удалять существующую базу данных и папку `storage/`.
4. Перезапустить или заново развернуть приложение на Bothost.

Ручные SQL-команды не требуются. Новая таблица фоновой очереди и миграция старого лимита создаются автоматически при запуске.

## Проверка после запуска
1. Открыть «Управление библиотекой» → «Импорт книг».
2. Отправить ZIP.
3. Дождаться сообщения «Архив принят».
4. Открыть другой раздел бота: он должен работать, пока сообщение импорта обновляет прогресс.

База, `.env`, пользовательское хранилище, кэш и тестовые материалы в архив не включены.
