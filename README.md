## Как загрузить на GitHub

1. Создайте репозиторий на GitHub.
2. Загрузите всё содержимое этой папки.
3. Не загружайте файл `.env`, используйте только `.env.example`.
4. На Bothost подключите репозиторий.
5. В переменных окружения укажите значения из `.env.example`.

## Переменные окружения

```env
BOT_TOKEN=токен_от_BotFather
OWNER_IDS=123456789
DATABASE_PATH=data/voxlyra.sqlite3
RUN_WEBAPP=true
PORT=3000
WEBAPP_URL=https://ваш-домен-на-bothost
CHANNEL_ID=@username_канала
BOT_USERNAME=username_бота_без_@
```

`BOT_USERNAME` нужен для кнопок покупки из Mini App: платная глава или аудио открывает бота с нужным счётом.

## Запуск локально

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python main.py
```

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python main.py
```

## Как автор загружает главы

1. Открывает `➕ Ещё`.
2. Нажимает `✍️ Автору`.
3. Открывает `📚 Мои книги`.
4. Выбирает книгу.
5. Нажимает `➕ Главы`.
6. Выбирает `➕ Ввести главу вручную` или `📄 Загрузить файл`.
7. При загрузке файла бот показывает предпросмотр.
8. Автор подтверждает сохранение.

## Как автор загружает аудио

1. Открывает `➕ Ещё`.
2. Нажимает `✍️ Автору`.
3. Открывает `📚 Мои книги`.
4. Выбирает книгу.
5. Нажимает `🎧 Аудио`.
6. Выбирает `➕ Добавить аудиоглаву` или `📦 Загрузить ZIP`.
7. Указывает диктора и цену.
8. Загружает MP3/M4A/OGG/WAV или ZIP с аудиофайлами.

## Как работает покупка

1. Читатель нажимает платную главу или аудио.
2. Бот отправляет счёт Telegram Stars.
3. После `successful_payment` покупка записывается в базу.
4. Пользователь получает доступ через кнопку в боте и через раздел `⭐ Моё`.
5. Доход автора попадает в удержание.
6. Комиссия платформы считается автоматически.

## Как работает возврат

1. Пользователь открывает `⭐ Моё`.
2. Выбирает покупку.
3. Нажимает `↩️ Запросить возврат`.
4. Пишет причину.
5. Владелец или модератор с правом `↩️ Возвраты` проверяет запрос.
6. При одобрении бот вызывает возврат Stars и помечает покупку как возвращённую.
7. Авторская запись по этой покупке тоже получает статус `refunded`.

## Как автор выбирает жанры и признаки книги

Там, где можно выбрать несколько вариантов, используется не ручной ввод, а отметки:

```text
▫️ Драма
✅ Тёмный хоррор
✅ Детектив
▫️ Триллер
✅ Готово
```

Так сделано для жанров, сюжетных тегов, аудитории и предупреждений. Автор руками пишет только то, что нельзя выбрать из справочника: название, описание, текст главы, имя диктора и похожие свободные поля.

## Реклама внутри чтения

В читалке есть нативный блок похожих книг. Он может показываться сверху главы, снизу главы или в двух местах сразу. Подбор идёт по совпадению жанров, сюжетных тегов и аудитории, которые автор отметил при создании книги.

Управление для владельца:

```text
👑 Управление → ⚙️ Настройки → 📖 Реклама в читалке
```

Обычные пользователи, авторы и модераторы без прав это меню не видят.

## Mini App и доступ

Mini App использует `window.Telegram.WebApp.initData`. Сервер проверяет подпись Telegram и только после этого отдаёт:

- купленные платные главы;
- купленные аудиофайлы;
- комментарии;
- сохранение прогресса чтения;
- сохранение прогресса прослушивания;
- закладки;
- отзывы и оценки.

Если открыть Mini App не из Telegram, платный доступ не выдаётся.

## Как автор выводит средства

1. Автор открывает `➕ Ещё → ✍️ Автору → 💰 Доход`.
2. Нажимает `🏦 Реквизиты` и один раз указывает способ выплаты.
3. Когда средства вышли из удержания, нажимает `📤 Запросить выплату`.
4. Бот проверяет минимальную сумму, активные заявки, заморозку выплат и доступный баланс.
5. Заявка появляется у владельца в `👑 Управление → 💰 Финансы → 📤 Заявки на выплату`.
6. Владелец проверяет автора, жалобы, возвраты и реквизиты.
7. После реального перевода владелец нажимает `✅ Отметить выплачено`.
8. Сумма переходит в статус `выплачено`, а действие пишется в журнал.

Выплаты на старте ручные. Это безопаснее: если автор загрузил чужой текст, получил жалобы или есть спорные возвраты, владелец может заморозить выплаты до проверки.

В скрытом меню владельца появился раздел `Финансы → Платёжные системы`. В нём отдельно включаются Stars, связка ЮKassa через BotFather, внешний контур ЮKassa, выплаты авторам, тестовый режим и защита содержимого. Ключи и токены можно внести позже через Mini App; они сохраняются зашифрованно.

Цифровой контент внутри Telegram по-прежнему продаётся только за Stars. Provider token ЮKassa из BotFather не подставляется в счета книг, глав, аудио или подписок. Подробная схема и точные поля описаны в `docs/PAYMENTS_AND_PROTECTION_V1_9_4.md`.

Если автор не разрешил скачивание, читалка блокирует обычное копирование и сохранение, не выдаёт файл и наносит персональный водяной знак. Если автор разрешил скачивание, ограничения снимаются и появляется загрузка доступного текста. Системный скриншот невозможно гарантированно заблокировать в браузерном Mini App, поэтому защита строится на затруднении копирования и прослеживаемости утечки.
