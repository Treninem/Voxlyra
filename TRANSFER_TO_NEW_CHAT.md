# Финальный тест-план Вокслиры

## 1. Проверка запуска

- Бот отвечает на `/start`.
- Главное меню появляется без ошибок.
- У владельца появляется `👑 Управление`.
- У обычного пользователя `👑 Управление` не появляется.
- `👑 Управление → 🧩 Система` показывает проверки.

## 2. Проверка автора

- Пользователь может зарегистрироваться автором.
- Псевдоним сохраняется.
- При новой книге псевдоним повторно не спрашивается.
- Книга создаётся в статусе черновика.
- Галочки жанров и тегов сохраняются.
- Автор может добавить главу вручную.
- Автор может загрузить файл книги.
- Автор может отправить книгу на проверку.

## 3. Проверка модерации

- Владелец может добавить администратора.
- Владелец может выдать одно право.
- Модератор видит только это право.
- Модератор может опубликовать книгу с главами.
- Модератор не может опубликовать книгу без глав.

## 4. Проверка Mini App

- `📚 Читать` открывает каталог.
- Страница книги открывается.
- Бесплатная глава открывается.
- Платная глава не отдаёт текст без покупки.
- После покупки доступ появляется.
- Прогресс чтения сохраняется.
- Закладка сохраняется.
- Отзыв сохраняется.
- Комментарий сохраняется.

## 5. Проверка аудио

- Автор может добавить аудиоглаву.
- Аудиоглава видна на странице книги.
- Бесплатное аудио воспроизводится.
- Платное аудио требует покупку.
- После покупки файл отдаётся через защищённый API.
- Прогресс прослушивания сохраняется.

## 6. Проверка платежей

- Счёт Stars создаётся.
- После successful_payment создаётся покупка.
- Авторский доход попадает в удержание.
- Возврат создаёт заявку.
- Владелец может одобрить возврат.

## 7. Проверка выплат

- Автор указывает реквизиты.
- Автор создаёт заявку на выплату.
- Владелец видит заявку.
- Владелец может одобрить, отклонить, заморозить и отметить выплату.

## 8. Проверка рекламы

- Автор создаёт рекламную кампанию.
- Автор пополняет бюджет рекламы Stars.
- Реклама показывается внутри главы.
- Клик записывается.
- В отчёте видны показы и клики.

## 9. Проверка жалоб и блокировок

- Пользователь может отправить жалобу.
- Владелец видит жалобу.
- Владелец может закрыть жалобу.
- Владелец может заблокировать пользователя.
- Владелец может заблокировать книгу.

## 10. Проверка канала

- Бот добавлен администратором канала.
- `CHANNEL_ID` указан правильно.
- При публикации книги бот отправляет пост.

## 11. Проверка уведомлений

- Общий переключатель отключает все пользовательские уведомления.
- Новые главы, аудиоглавы и скидки управляются отдельно.
- Автор не получает читательское уведомление о собственной книге.
- Повторная обработка события не создаёт дубликат сообщения.

## 12. Финальная проверка аватаров и релиза

- `voxlyra_bot_avatar_final.png` открывается и подходит для круглой обрезки.
- `voxlyra_channel_avatar_final.png` открывается и подходит для круглой обрезки.
- Встроенные копии в `static/img/` совпадают с отдельными файлами.
- Аватары читаются в размере 64 пикселя.
- Полный архив не содержит `.env`, рабочую базу, кэш и временные файлы.
- Архив обновления не заменяет `data/voxlyra.sqlite3` и содержимое `storage/`.
- После Redeploy отвечают `/health`, `/readiness`, `/start`, каталог и скрытая панель владельца.
